# Changelog

All notable changes to this project are documented in this file.
The format is based on [Keep a Changelog](https://keepachangelog.com/).

## [Unreleased]

### Added
- **tsc typecheck gate** in CI (`npm run typecheck`) — catches type errors
  in source and test files before they reach the build.
- **Bundle-size guard** in CI (`npm run size-check`) — fails if the gzipped
  core bundle exceeds the 100KB PDD budget.
- **`prepublishOnly` script** — rebuilds `dist/` before `npm publish` so a
  stale tree can't ship outdated artifacts.
- **Container-scoped overlay** — `MascotConfig.container` scopes the overlay
  to an element instead of the full viewport (position: absolute + ResizeObserver).
  The playground now renders the mascot inside its preview box.
- **Accessible speech bubbles** — bubble div has `role="status"` +
  `aria-live="polite"` so screen readers announce `engine.say()` text.
- **Keyboard-accessible drag** — when `draggable` is enabled, the canvas gets
  `tabindex=0` and arrow keys (Shift+arrow for larger steps) emit drag events.
- **`device.lost` handler** in `WebGPURenderer` — tears down on GPU device
  loss and calls an optional `onDeviceLost(reason)` callback for fallback.
- **`CHANGELOG.md`** — this file.

### Changed
- **`auto-init` marked as having side effects** in `package.json` so bundlers
  don't tree-shake away the custom-element definition.

### Removed
- **`FrameTimer.setFps`** — dead code with no callers after the React Native
  runtime stopped using it.
- **`example/tiny-mascot.global.js`** from git tracking — it's a build
  artifact regenerated by `npm run example` and the pages workflow.

### Fixed
- Test mock type errors that blocked `tsc --noEmit` (GPUCanvasContext overload
  casts, untyped `this` in mock `use` functions, wrong relative import path).

## Post-MVP Expansion

### Added
- **Zero-config auto-init** (`mascot-plugin/auto-init`) — defines the
  `<tiny-mascot>` custom element and exposes a `TinyMascot` global. Built-in
  default mascot works with no spritesheet or JavaScript.
- **CI workflow** — lint, build, and test on push and PR.
- **Spritesheet packer CLI** (`mascot-pack`) — packs PNG frames, Aseprite
  JSON, and GIFs into a uniform-grid spritesheet + metadata. Supports
  `--idle`/`--react` animation ranges.
- **Interactive playground** — live mascot preview with code generation
  (HTML + React tabs). Deployed to GitHub Pages.
- **Accessibility** — reduced-motion mode (freezes animation), visibility
  pause (stops when tab is hidden), ARIA label on the canvas.
- **MascotManager** (`mascot-plugin/manager`) — spawn, control, and tear down
  multiple mascots on one page with bulk `emitAll`/`stopAll`.
- **Pointer drag-to-move** — `draggable` config option with viewport clamping
  and `teleport`/`releasePosition` API.
- **Speech bubbles** — `engine.say(text, durationMs)` renders a bubble above
  the mascot via the overlay.
- **Unity bridge** (`mascot-plugin/unity`) — JSON-over-stdio protocol for
  running the mascot engine as a subprocess controlled by a game engine.
- **React Native bridge** (`mascot-plugin/react-native`) —
  `ReactNativeRuntime` + `SkiaRenderer` for native mascot rendering.
- **MediaRecorder recording** (`MascotRecorder`) — records the mascot canvas
  to a video Blob with MIME-type fallback.
- **WebGPU renderer** (`mascot-plugin/webgpu`) — GPU-accelerated sprite
  rendering with premultiplied-alpha blend and ImageBitmap lifecycle management.
- **Package metadata** — optional peer deps, `sideEffects`, keywords, LICENSE.

### Fixed
- Overlay/engine cleanup on error paths (asset load failure, `start()` throw).
- GIF disposal methods (0–3) in the packer's frame compositor.
- ESM-safe `require` for `pngjs`/`omggif` via `createRequire`.
- Premultiplied-alpha blend in the WebGPU renderer.
- User input escaping in the playground code preview (self-XSS prevention).
- Canvas positioning and sizing per `RenderFrame` in the WebGPU renderer.
- `ImageBitmap` closed after GPU upload to prevent memory leaks.
